---
title: "Rate Limits"
description: "Configure per-IP sliding-window rate limiting and auth lockout protection for Argus MCP."
---

The `rate_limits:` block configures a per-IP sliding-window rate limiter applied to all incoming requests, with an auth lockout mechanism that temporarily blocks IP addresses after repeated authentication failures.

```yaml
rate_limits:
  enabled: true
  default:
    requests: 100
    window_seconds: 60
  auth_lockout_threshold: 5
  auth_lockout_window_seconds: 300
  auth_lockout_duration_seconds: 900
```

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `rate_limits.enabled` | bool | `true` | Enable per-IP sliding window rate limiting |
| `rate_limits.default.requests` | int | `100` | Maximum requests allowed per window. Range: 1–100000. |
| `rate_limits.default.window_seconds` | int | `60` | Sliding window size in seconds. Range: 1–86400. |
| `rate_limits.auth_lockout_threshold` | int | `5` | Number of auth failures (401/403 responses) before an IP is locked out. Range: 1–1000. |
| `rate_limits.auth_lockout_window_seconds` | int | `300` | Time window in seconds for counting auth failures. Range: 10–86400. |
| `rate_limits.auth_lockout_duration_seconds` | int | `900` | Duration in seconds an IP remains locked out. Range: 10–86400. |

## Default Rate Limit

The `default` block sets the sliding-window limit applied to every IP address. With the defaults, each IP can send up to 100 requests per 60-second window. Requests exceeding the limit receive HTTP 429.

Example: reduce the limit for a low-traffic deployment:

```yaml
rate_limits:
  default:
    requests: 30
    window_seconds: 60
```

The rate limiter uses a TTLCache with bounded memory — it will not grow unboundedly even under sustained traffic from many unique IPs.

## Auth Lockout

The auth lockout mechanism tracks 401 and 403 responses per IP. After `auth_lockout_threshold` failures within `auth_lockout_window_seconds`, the IP is blocked for `auth_lockout_duration_seconds`. All requests from a locked-out IP receive HTTP 429 until the lockout expires.

**Default behavior:** 5 auth failures within 5 minutes → 15-minute lockout.

This protects against brute-force token guessing. Example: tighten the lockout for a high-security deployment:

```yaml
rate_limits:
  auth_lockout_threshold: 3
  auth_lockout_window_seconds: 120
  auth_lockout_duration_seconds: 3600
```

This locks out an IP for 1 hour after 3 failed auth attempts within 2 minutes.
