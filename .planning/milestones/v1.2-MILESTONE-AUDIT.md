### Key Integration Points

- `lib/catalog.ts fetchAllCatalogEntries()` reuses existing `fetchCatalogIndex()` + `fetchCategoryConfigs()` — no new Octokit client, no new cache layer. The `CatalogEntry.category` field plumbed at the data layer enables category sidebar, card badges, and `categoryLabels` mapping in CatalogBrowser.
- `CatalogBrowser.tsx` (`'use client'`) wrapped in `<Suspense>` in `app/docs/catalog/page.tsx` (Server Component) — satisfies both Next.js static export requirement for `useSearchParams` and the App Router Server/Client Component boundary.
- `settings/navigation.ts` Catalog href changed from external GitHub URL (`external: true`) to internal `/docs/catalog` — Next.js client-side routing now handles the Catalog nav item like all other top-nav links.
- `components/ui/badge.tsx` → `CatalogEntryCard.tsx` → `CatalogGrid.tsx` → `CatalogBrowser.tsx` → `app/docs/catalog/page.tsx`: Clean one-direction import chain; no circular dependencies.
